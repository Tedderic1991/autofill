**lib/presentation/profiles/profile_edit_screen.dart:**
A `ConsumerStatefulWidget` that handles both create (profileId == null) and edit (profileId != null).
Form fields (all using `TextFormField` unless noted):
- displayName (required, validated non-empty)
- dateOfBirth (optional, `TextFormField` with date format hint OR `showDatePicker`)
- addressLine1, addressLine2, city, stateProvince, postalCode, country (all optional TextFormField)
- phone (optional)
- allergies (optional, multi-line TextFormField)
- emergencyContactName, emergencyContactPhone (optional)
- relationshipTag: `DropdownButtonFormField<RelationshipTag>` with parent/child/guardian options
- Avatar: `CircleAvatar` + `TextButton('Pick photo')` that calls `ImagePicker().pickImage()`
  compressed to 512x512 / 85% quality, saves file to `${documentsDir}/avatars/${uuid}.jpg`,
  stores path in a local state variable (written to profile on save)

On submit (save button in AppBar):
1. Validate form (`_formKey.currentState!.validate()`)
2. Call `profileUseCase.createProfile(request)` or `updateProfile(id, request)`
3. Catch `ProfileLimitReached` → `context.go('/paywall')` (navigate to paywall stub)
4. On success → `Navigator.pop(context)` (back to list)
5. On error → show SnackBar with error message

Custom fields section at the bottom of the form:
- If editing an existing profile: load custom fields via `customFieldUseCaseProvider`
- Embed `CustomFieldEditorWidget(profileId: profile.id)` below built-in fields

**lib/presentation/profiles/custom_field_editor_widget.dart:**

  - "Add" button calls `customFieldUseCase.addField(profileId, label, type)`
- Edit field: same bottom sheet pre-populated; "Save" calls `customFieldUseCase.editField(...)`
