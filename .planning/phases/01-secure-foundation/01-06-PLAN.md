A `ConsumerStatefulWidget`. On `initState`, immediately call `_triggerAuth()` so the
biometric prompt appears automatically when the screen mounts (no extra tap required).
Also show an "Unlock" button in the body so the user can retry after dismissing.

`_triggerAuth()` method:
```dart
Future<void> _triggerAuth() async {
  final auth = LocalAuthentication();
  // Check capabilities first (pitfall prevention)
  final canCheck = await auth.canCheckBiometrics;
  final isSupported = await auth.isDeviceSupported();

  if (!isSupported) {
    // Device has no lock screen at all — allow access (edge case)
    ref.read(authStateNotifierProvider.notifier).unlock();
    return;
  }

  try {
    final authenticated = await auth.authenticate(
      localizedReason: 'Unlock your family profiles',
      options: const AuthenticationOptions(
        biometricOnly: false,  // REQUIRED — enables PIN/pattern fallback
        stickyAuth: true,
      ),
    );
    if (authenticated) {
      ref.read(authStateNotifierProvider.notifier).unlock();
    }
  } on PlatformException catch (e) {
    switch (e.code) {
      case 'NotEnrolled':
        // Show a SnackBar + guidance to set up biometrics/PIN
        _showError('Set up a screen lock in device Settings to use this app.');
        break;
      case 'LockedOut':
        _showError('Too many failed attempts. Use your device PIN to unlock.');
        break;
      case 'PermanentlyLockedOut':
        _showError('Biometrics locked. Open device Settings > Security to reset.');
        break;
      default:
        _showError('Authentication failed. Tap Unlock to try again.');
    }
  }
}
```

Do NOT use a single generic `catch` — each error code has a specific message
that guides the user to resolution (pitfall prevention from PITFALLS.md Pitfall 7).

Route configuration:
```dart
final _router = GoRouter(
  initialLocation: '/auth',
  redirect: (context, state) {
    // Read auth state from Riverpod outside a widget — use a global container
    // OR pass a ref via refreshListenable pattern
    final isLocked = ...; // see below
    final isAuthRoute = state.matchedLocation == '/auth';
    if (isLocked && !isAuthRoute) return '/auth';
    if (!isLocked && isAuthRoute) return '/profiles';
    return null;
  },
  routes: [
    GoRoute(path: '/auth', builder: (_, __) => const AuthGateScreen()),
    GoRoute(path: '/profiles', builder: (_, __) => const ProfileListScreen()),
    GoRoute(path: '/profiles/new', builder: (_, __) => const ProfileEditScreen()),
    GoRoute(
      path: '/profiles/:id/edit',
      builder: (_, state) => ProfileEditScreen(profileId: state.pathParameters['id']),
    ),
    GoRoute(path: '/paywall', builder: (_, __) => const PaywallStubScreen()),
  ],
);
```

For the auth redirect, use GoRouter's `refreshListenable` with a `ProviderListenable`
adapter. The cleanest approach for Riverpod + go_router:
```dart
// In AutofillApp.build():
final authState = ref.watch(authStateNotifierProvider);
final router = useMemoized(() => GoRouter(
  refreshListenable: ...,  // or rebuild router when auth changes
  ...
));
```

A simpler, correct approach: declare the GoRouter inside `ConsumerWidget.build()` using
a `ref.watch(authStateNotifierProvider)` to read the current state, and pass it as a
captured variable in the redirect closure. GoRouter will re-evaluate redirect whenever
the widget rebuilds (which happens when auth state changes via Riverpod).
